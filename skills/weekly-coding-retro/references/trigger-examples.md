# Trigger Examples

Use these prompts to test whether `weekly-coding-retro` triggers correctly.

## Positive (Chinese)

- 帮我做本周编码复盘：我有 commits、PR 评论和几个 bug 记录，想找出重复问题模式。
- 这周总在返工，帮我整理根因并给出下周 1-3 个改进实验。
- 根据我这周的 review feedback，生成个人能力提升复盘和行动项。
- 做一次 weekly coding retro，重点看调试效率和测试设计问题。

## Positive (English)

- Help me do a weekly coding retrospective using my commits, review comments, and bug notes.
- Summarize my recurring engineering mistakes this week and propose 1-3 improvement experiments.
- Turn this week of PR feedback into a personal coding improvement retro.
- I want a weekly coding retro focused on debugging speed and test quality patterns.

## Negative / Near Miss

- Give me a project status update for this sprint. (Status reporting, not personal skill retro)
- Plan next week's feature roadmap. (Planning deliverables, not retrospective learning)
- Write my annual performance review. (HR/performance review, not weekly coding retrospective)
- Summarize this single incident postmortem. (One incident summary, not recurring weekly pattern analysis)
